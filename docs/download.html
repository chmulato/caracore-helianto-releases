Download e Deploy — Helianto Condominium

Infraestrutura pronta para VPS e Nuvem (Docker Compose, Caddy e Nginx)

O sistema é implantado de forma autônoma utilizando contêineres Docker, garantindo a sua soberania de dados e facilidade de manutenção.

Como subir em produção (Docker Compose)

Os arquivos prontos de infraestrutura estão disponíveis para download direto do nosso repositório de releases no GitHub. Você também pode copiar as instruções básicas de deploy abaixo:

Fluxo de Deploy Rápido:

  1. Instale o Docker e o Docker Compose em sua VPS Linux.
  2. Baixe o template docker-compose.prod.yml.
  3. Configure as variáveis de ambiente necessárias em um arquivo .env local.
  4. Inicie os serviços: docker compose -f docker-compose.prod.yml up -d

Abaixo estão os templates de proxy e infraestrutura do ecossistema:

docker-compose.prod.yml Caddyfile (SSL Autônomo) nginx.conf (Proxy reverso)

Requisitos do Servidor (Recomendado)

  • Sistema Operacional: Ubuntu Server 22.04 LTS ou superior (64-bit)
  • Mínimo: 1 vCPU, 2 GB RAM (para uso leve / homologação)
  • Recomendado: 2 vCPUs, 4 GB RAM (operações ativas em produção)
  • Armazenamento: 20 GB de espaço SSD
  • Software: Docker Engine 24.0+ e Docker Compose v2.0+ instalado

Verificação e Segurança

O back-end do Helianto Condominium utiliza segurança ativa de banco de dados e higienizador de logs em tempo real (PrivacyGuard - LGPD Shield). Certifique-se de que a porta 5432 do banco PostgreSQL não seja exposta publicamente na internet (os templates de Docker fornecidos já asseguram essa proteção).
